## FaceSmashConfig

Pass a configuration object when creating a client:

```typescript
import { createFaceSmash } from '@facesmash/sdk';

const client = createFaceSmash({
  apiUrl: 'https://api.facesmash.app',
  modelUrl: 'https://cdn.jsdelivr.net/npm/@vladmandic/face-api/model',
  matchThreshold: 0.45,
  minQualityScore: 0.2,
  minDetectionConfidence: 0.3,
  maxTemplatesPerUser: 10,
  debug: false,
});
```

Or with the React provider:

```tsx
<FaceSmashProvider config={{
  apiUrl: 'https://api.facesmash.app',
  matchThreshold: 0.45,
  debug: true,
}}>
```

## All Options

| Option | Type | Default | Description |
|---|---|---|---|
| `apiUrl` | `string` | `https://api.facesmash.app` | PocketBase API URL for your backend |
| `modelUrl` | `string` | jsdelivr CDN | URL to load face-api.js models from |
| `minDetectionConfidence` | `number` | `0.3` | SSD MobileNet minimum confidence (0-1) |
| `matchThreshold` | `number` | `0.45` | Similarity threshold for face matching (0-1) |
| `minQualityScore` | `number` | `0.2` | Minimum quality score to accept a face scan (0-1) |
| `maxTemplatesPerUser` | `number` | `10` | Maximum face templates stored per user |
| `debug` | `boolean` | `false` | Enable console logging |

## Understanding Thresholds

### matchThreshold

Similarity scores range from 0 (completely different) to 1 (identical):

| Score Range | Meaning |
|---|---|
| `0.80 - 1.00` | Almost certainly the same person |
| `0.60 - 0.80` | Very likely the same person |
| `0.45 - 0.60` | Probably the same person (default match zone) |
| `0.30 - 0.45` | Uncertain — could be same or different |
| `0.00 - 0.30` | Almost certainly different people |

<Callout type="warn">
  Setting `matchThreshold` too low (below 0.35) increases false positives — strangers may be incorrectly identified.
  Setting it too high (above 0.65) increases false negatives — legitimate users may fail to login.
  The default of 0.45 with adaptive thresholds provides the best balance.
</Callout>

### Adaptive Thresholds

The SDK automatically adjusts the threshold based on conditions:

- **Poor lighting** (score < 0.4): threshold is lowered by up to 0.05 to compensate
- **Good lighting** (score > 0.8): threshold is raised slightly for stricter matching
- **User confidence boost**: experienced users with many successful logins get a small threshold reduction

### minQualityScore

Quality scoring combines detection confidence, lighting analysis, face size relative to the frame, and head pose (yaw/pitch/roll). A score of 0.2 is deliberately lenient to work in challenging conditions. Raise to 0.4–0.5 for higher security.

### minDetectionConfidence

This is the SSD MobileNet detection confidence. If the primary SSD detector fails to find a face above this threshold, the SDK falls back to TinyFaceDetector automatically.

## Events

Subscribe to SDK events for real-time feedback:

```typescript
const unsubscribe = client.on((event) => {
  console.log(event.type, event);
});
```

| Event | Payload |
|---|---|
| `models-loading` | `{ progress: number }` |
| `models-loaded` | — |
| `models-error` | `{ error: string }` |
| `face-detected` | `{ analysis: FaceAnalysis }` |
| `face-lost` | — |
| `login-start` | — |
| `login-success` | `{ user: UserProfile, similarity: number }` |
| `login-failed` | `{ error: string, bestSimilarity?: number }` |
| `register-start` | — |
| `register-success` | `{ user: UserProfile }` |
| `register-failed` | `{ error: string }` |

## PocketBase Backend

The SDK connects to a [PocketBase](https://pocketbase.io) instance. You can access the underlying PocketBase client directly:

```typescript
const client = createFaceSmash({ apiUrl: 'https://your-api.example.com' });
client.pb; // PocketBase instance — for custom queries
```

### Required Collections

Your PocketBase instance needs these collections:

| Collection | Key Fields |
|---|---|
| `user_profiles` | `name`, `email`, `face_embedding` (JSON array of 128 floats) |
| `face_templates` | `user_email`, `descriptor` (JSON array), `quality_score`, `label` |
| `face_scans` | `user_email`, `face_embedding`, `confidence`, `scan_type`, `quality_score` |
| `sign_in_logs` | `user_email`, `success` |

## Self-Hosting

To use FaceSmash with your own backend instead of `api.facesmash.app`:

1. Install PocketBase on your server
2. Create the collections listed above
3. Point the SDK to your PocketBase URL:

```typescript
const client = createFaceSmash({
  apiUrl: 'https://your-pocketbase-instance.com',
});
```
