## Overview

If the pre-built `<FaceLogin>` and `<FaceRegister>` components don't fit your design, you can use the low-level SDK to build a fully custom face capture experience.

## Architecture

```
Your UI Layer
    │
    ▼
FaceSmash Core SDK
    ├── initModels()        → Load neural networks
    ├── detectFace()        → Find face in frame
    ├── extractDescriptor() → Get 128-d vector
    ├── analyzeQuality()    → Score lighting, pose, sharpness
    └── matchDescriptors()  → Compare against stored faces
```

## Step 1: Set Up Camera

Use the browser's `getUserMedia` API to access the camera:

```typescript
async function setupCamera(videoElement: HTMLVideoElement) {
  const stream = await navigator.mediaDevices.getUserMedia({
    video: {
      facingMode: 'user',
      width: { ideal: 640 },
      height: { ideal: 480 },
    },
  });
  videoElement.srcObject = stream;
  await videoElement.play();
}
```

## Step 2: Detection Loop

Run face detection on each frame:

```typescript
import { initModels, detectFace } from '@facesmash/sdk/core';

await initModels();

let animationId: number;

function startDetectionLoop(video: HTMLVideoElement) {
  async function loop() {
    const detection = await detectFace(video);

    if (detection) {
      const { box, score } = detection;
      drawBoundingBox(box);          // Your custom overlay
      updateConfidence(score);        // Your custom UI
    } else {
      clearOverlay();
    }

    animationId = requestAnimationFrame(loop);
  }

  loop();
}

function stopDetectionLoop() {
  cancelAnimationFrame(animationId);
}
```

## Step 3: Capture and Analyze

When you want to capture a face (e.g., on button click or auto-trigger):

```typescript
import { extractDescriptor, analyzeQuality } from '@facesmash/sdk/core';

async function captureFace(video: HTMLVideoElement) {
  const detection = await detectFace(video);
  if (!detection) throw new Error('No face detected');

  // Extract the 128-dimensional face descriptor
  const descriptor = await extractDescriptor(video, detection);

  // Analyze quality
  const quality = analyzeQuality(detection, video);

  return {
    descriptor,                    // Float32Array(128)
    quality: quality.overallScore, // 0-1
    lighting: quality.lightingScore,
    sharpness: quality.sharpness,
  };
}
```

## Step 4: Match or Register

```typescript
import { matchDescriptors } from '@facesmash/sdk/core';

// For login — match against stored faces
async function loginWithFace(descriptor: Float32Array) {
  const storedFaces = await fetch('/api/faces').then(r => r.json());

  const result = matchDescriptors(descriptor, storedFaces, {
    threshold: 0.45,
    adaptiveThreshold: true,
  });

  if (result.isMatch) {
    return { success: true, user: result.matchedUser };
  }
  return { success: false, bestSimilarity: result.bestSimilarity };
}

// For registration — store the face
async function registerFace(email: string, descriptor: Float32Array) {
  await fetch('/api/faces/register', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      email,
      descriptor: Array.from(descriptor),
    }),
  });
}
```

## Custom Overlay Example

Draw a face bounding box with quality indicators:

```typescript
function drawBoundingBox(
  ctx: CanvasRenderingContext2D,
  box: { x: number; y: number; width: number; height: number },
  quality: number
) {
  // Color based on quality
  const color = quality > 0.6 ? '#22c55e'   // green
              : quality > 0.35 ? '#eab308'   // yellow
              : '#ef4444';                     // red

  ctx.strokeStyle = color;
  ctx.lineWidth = 3;
  ctx.strokeRect(box.x, box.y, box.width, box.height);

  // Quality label
  ctx.fillStyle = color;
  ctx.font = '14px Inter, sans-serif';
  ctx.fillText(
    `Quality: ${(quality * 100).toFixed(0)}%`,
    box.x,
    box.y - 8
  );
}
```

## Tips

- **Debounce detection** — Don't run detection on every frame. 3-5 FPS (200-300ms intervals) is sufficient and saves battery.
- **Pre-warm models** — Call `initModels()` early (e.g., on app load) so models are ready when the user reaches the login page.
- **Capture multiple frames** — Average descriptors from 3-5 good captures for more reliable matching.
- **Show quality feedback** — Guide users to improve lighting, face angle, and distance for better results.
